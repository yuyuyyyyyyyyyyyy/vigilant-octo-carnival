# 多棱镜 · 棱镜事件解释器 (Multi-Prism Interpreter)

> 用多个"棱镜"重新看见你经历的事件——一个把单一结论拆成多视角解释的实验性产品。

## 这是什么

多棱镜是一个 Web 应用：你输入一段自己经历的人际 / 生活事件，它从**认知心理学、社会心理学、系统论、哲学**等多个跨学科视角生成多角度解释，并进一步给出**可验证的实验设计**——帮你用一个小实验检验哪个解释更接近真相，而不是只丢一个结论给你。

**核心理念**：人对事件的"理解"往往是单一叙事的产物。多棱镜的价值不在于告诉你"答案"，而在于让你看见"原来还能这样看"。

## 核心功能

- **双模式引擎**：自动判断意图，在「发散理解（Explore）」与「收敛判断（Converge）」之间切换——前者打开多种可能性，后者给出当前最优的临时结论。
- **多棱镜解释**：每个事件从多个跨学科视角生成解释，每个解释带可信度评分（1-5）与"适用边界 / 不成立条件"。
- **事实 vs 推测分离**：严格区分用户输入的「事实」与模型的「推测」，拒绝心理标签（如"高敏感人格"），只解释机制。
- **巴纳姆效应规避**：内置约束——"若一句解释适用于 80% 以上的人则判失败，重新推导"，强迫解释具体到"为什么偏偏是这件事 / 这个人 / 现在"。
- **变量分析**：拆出影响事件的关键变量（认知 / 人际 / 历史 / 外部等），解释每个变量的作用。
- **可验证实验设计**：为每个解释给出可操作的验证实验（怎么做 / 预期变化 / 判断标准 / 模型如何更新），把"观点"变成"可检验的假设"。
- **历史与复盘**：保存每次分析，可回看思考轨迹；多次解释不重复同一框架。
- **分享卡**：把分析结果生成图片卡片，便于记录与分享。
- **匿名访问统计**：基于 Netlify Blobs 的聚合访问看板（/stats），不含任何个人数据。

## 技术栈

| 层 | 技术 |
|----|------|
| 全栈框架 | Next.js 14（App Router） |
| 前端 | React 18 + TypeScript |
| 样式 / 动效 | Tailwind CSS + framer-motion |
| 分享卡生成 | html-to-image |
| 存储 / 统计 | @netlify/blobs |
| LLM 接入 | 环境变量配置（`LLM_API_KEY` / `LLM_BASE_URL` / `LLM_MODEL`），兼容 OpenAI / DeepSeek 等任意 Chat Completions 接口 |

## 快速开始

```bash
# 1. 安装依赖
npm install

# 2. 配置环境变量（可选）
#    未配置时应用会自动降级到内置「模拟解释引擎」，仍可完整体验产品逻辑
cp .env.example .env.local   # 然后填入 LLM_API_KEY=xxx
# 或直接：
echo "LLM_API_KEY=your_key_here" > .env.local

# 3. 启动
npm run dev
# 打开 http://localhost:3000
```

> 容错设计：无 API Key 时自动使用内置预设解释库（mock 引擎），保证产品在任何环境下都能跑通——这是有意设计的降级方案，而非缺陷。

## 产品理念与我（PM 视角）

> 这一节写给面试官：本项目从概念到落地由我独立驱动。

- **起点**：观察到"人被单一叙事困住"的普遍状态（例如认定"某件事证明我差"），想做一个工具让人看见同一件事的多种合理解释。
- **我的角色**：产品定义、多视角框架设计、与 AI 协作迭代 Prompt（v1 → v5 融合版）、功能优先级决策、双模式与字段结构迭代。
- **关键决策**：坚持"不给单一答案，只给多视角 + 可验证实验"——这是产品的灵魂，也体现我对"AI 该不该替人下结论"的判断：不替人定义，只增加理解的可能性。
- **工程权衡**：为演示环境加入 mock 降级引擎，并把 LLM 接入抽象为环境变量，使产品既能跑真实模型，也能无 key 演示。
- **诚实边界**：当前内置解释库为预设模板，真实 LLM 接入后能力显著增强；统计看板为匿名聚合，不含个人数据。
